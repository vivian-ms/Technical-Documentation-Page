Introduction

CSS Grid Layout is a new layout model for CSS that has powerful abilities to control the sizing and positioning of boxes and their contents. In the grid layout model, the children of a grid container can be positioned into arbitrary slots in a predefined flexible or fixed-size layout grid.

When grid is combined with other parts of CSS such as flexbox, you can create layouts that were previously impossible to build in CSS.

A grid is an intersecting set of horizontal and vertical lines - one set defining columns and the other set defining rows. Elements can be placed onto the grid within these column and row lines.

Fixed and flexible track sizes

A grid can be created with fixed or flexible track sizes. A grid track is a generic term for either a grid column or a grid row. For example, using pixels creates a grid with fixed track sizes and using percentages or the new fr unit creates a grid with flexible track sizes.

Item placement

Items can be placed into a precise location on the grid using line numbers, names or by targeting an area of the grid. Grid also contains an algorithm to control the placement of items not given an explicit position on the grid.

Creation of additional tracks to hold content

Explicit grid can be defined with grid layout. The Grid Layout specification is flexible enough to add additional rows and columns when needed. Features such as adding "as many columns that will fit into a container" are included.

Alignment control

Grid contains alignment features to control how the items align once placed into a grid area and how the entire grid is aligned.

Control of overlapping content

More than one item can be placed into a grid cell or area and they can partially overlap each other. The z-index property can be used to control the overlap.

Grid Container

A grid starts with the grid container. A grid container is created by declaring display: grid or display: inline-grid on an element. All the direct children of that element becomes grid items. In the example below, .wrapper is a grid container and its children are grid items. 

<div class="wrapper">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
</div>
.wrapper {
  display: grid;
}

Using only the border, border-radius, background-color, padding, and color properties on the grid container and grid items, the code above will be displayed like the image below.

Grid Tracks

Rows and columns of a grid are defined with the grid-template-columns and grid-template-rows properties.

Using the code above, we can add the grid-template-columns property to create a grid with three 200-pixel-wide column tracks. Each child item will be automatically placed in each grid cell. 

<div class="wrapper">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
</div>
.wrapper {
  display: grid;
  grid-template-columns: 200px 200px 200px;
}

The fr Unit

Tracks can be defined using any length unit. Grid also introduces an additional length unit to help create flexible grid tracks. The new fr unit represents a fraaction of the available space in the grid container. The example below uses fr to create three equal width tracks that grow and shrink according to the available space. 

<div class="wrapper">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
</div>
.wrapper {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
}

In the following example, the available space is split into four, where two parts are given to the first track and one part each to the next two tracks. 

.wrapper {
  display: grid;
  grid-template-columns: 2fr 1fr 1fr;
}

The example below mixes absolute sized tracks with fraction units. A fixed width of 500px is taken for the first track and the remaining space is divided into three and assigned in proportion to the two flexible tracks. 

.wrapper {
  display: grid;
  grid-template-columns: 500px 1fr 2fr;
}

Track listings with repeat() notation

Large grid with many tracks can use the repeat() notation, to repeat all or a section of the track listing. For example, 

.wrapper {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
}

Can also be written as:

.wrapper {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

Repeat notation can be used for a part of the track listing. In the example below, the grid has an initial 20px track, a repeating section of 6 1fr tracks, and a final 20px track. 

.wrapper {
  display: grid;
  grid-template-columns: 20px repeat(6, 1fr) 20px;
}

Repeat notation takes the track listing and uses it to create a repeating pattern of tracks. In the example below, the grid will consist of 10 tracks: a 1fr track followed by a 2fr track repeated five times. 

.wrapper {
  display: grid;
  grid-template-columns: repeat(5, 1fr, 2fr);
}

The implicit and explicit grid

The grid examples above used the grid-template-columns property to define the column tracks but the grid also created rows on its own. Those rows are part of the implicit grid. Explicit grid consists of any rows and columns defined with grid-template-columns or grid-template-rows.

If something is placed outside of the defined grid - or due to the amount of content, more grid tracks are needed - then the grid creates rows and columns in the implicit grid. These tracks will be auto-sized by default, resulting in their size being based on the content inside of them.

To set the size for tracks created in the implicit grid, the grid-auto-rows and grid-auto-columns properties can be used.

The example below uses grid-auto-rows to ensure that tracks created in the implicit grid are 200px tall. 

<div class="wrapper">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
</div>
.wrapper {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: 200px;
}

Track sizing and minmax()

When setting up an explicit grid or defining the size for automatically created rows or columns, it may be preferable to give tracks a minimum size but also ensure they expand to fit any content that is added. For example, you may want rows to never collapse smaller than 100px but if the content stretches to 300px in height, the row should stretch to that height.

Grid has a solution for this with the minmax() function. The example below uses minmax() in the value of grid-auto-rows. It will automatically create rows with a minimum height of 100px and a maximum of auto. Using auto means that the height of that row will be the height of the cell with the tallest item. 

<div class="wrapper">
  <div>One</div>
  <div>Two
    <p>I have some more content in.</p>
    <p>This makes me taller than 100 pixels.</p>
  </div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
</div>
.wrapper {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: minmax(100px, auto);
}

Grid Lines

When we define a grid, we define the grid tracks, not the lines. Grid will then give us numbered lines to use when positioning items. In a three column, two row grid, there are four column lines.

Lines are numbered according to the writing mode of the document. In a left-to-right language, line 1 is on the left-hand side of the grid. In a right-to-left language, line 1 is on the right-hand side of the document. Lines can also be named, which will be shown later in the documentation.

Positioning items against lines

When we place an item, we target the line, rather than the track.

The example below uses grid-column-start, grid-column-end, grid-row-start, and grid-row-end properties to place the first two items in the three column track grid. Working from left to right, the first item is placed against column line 1 and spans to column line 4, which is the far-right line on the grid. It begins at row line 1 and ends at row line 3, spanning two row tracks.

The second item starts on grid column 1 and spans one track. This is the default so there's no need to specify the end line. It also spans two row tracks from row line 3 to row line 5. The other items will automatically place themselves into empty spaces on the grid. 

<div class="wrapper">
  <div class="box1">One</div>
  <div class="box2">Two</div>
  <div class="box3">Three</div>
  <div class="box4">Four</div>
  <div class="box5">Five</div>
</div>
.wrapper {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: 100px;
}

.box1 {
  grid-column-start: 1;
  grid-column-end: 4;
  grid-row-start: 1;
  grid-row-end: 3;
}

.box2 {
  grid-column-start: 1;
  grid-row-start: 3;
  grid-row-end: 5;
}

Grid Cells

A grid cell is the smallest unit on a grid. Conceptually, it's like a table cell. As seen in the previous examples, once a grid is defined as a parent, the child items will lay themselves out in one cell each of the defined grid. The image below highlights the first cell of the grid.

Grid Areas

Items can span one or more cells both by rows or by columns and this creates a grid area. Grid areas must be rectangular, e.g. it's not possible to create an L-shaped area. The highlighted grid area below spans two row and two column tracks.

Gutters

Gutters or alleys between grid cells can be created using the column-gap and row-gap properties, or the shorthand gap property. The example below has a 10px gap between columns and a 1em gap between rows. 

<div class="wrapper">
  <div>One</div>
  <div>Two</div>
  <div>Three</div>
  <div>Four</div>
  <div>Five</div>
</div>
.wrapper {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  column-gap: 10px;
  row-gap: 1em;
}

Note: When grid first shipped in browsers, the column-gap, row-gap, and gap were prefixed with the grid- prefix as grid-column-gap, grid-row-gap, and grid-gap respectively.

Browsers are updating to remove this prefix but the prefixed versions will be maintained as aliases making them safe to use. At the present time, some browsers don't support the unprefixed versions.

Any space used by gaps will be accounted for before space is assigned to the flexible length fr tracks. Gap act for sizing purposes like a regular grid track but you can't place anything into a gap. In terms of line-positioning, the gap acts like a fat line.
